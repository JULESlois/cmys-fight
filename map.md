# 历史实现规格：主基地大地图化

> 该规格的大部分内容已经在当前 Hub 系统中实现。本文保留为设计记录；实际行为以 `src/game/hub`、`src/game/world`、`HubState.ts` 和对应 smoke 测试为准。

对 cmys-fight 进行一次“主基地大地图化”重构。

目标：
将当前偏菜单式 / 页面式的基地与前端流程，重构为一个“西幻风格的大型可滚动主基地地图（Hub Map）”。
注意：本轮重点是“基地地图系统”和“Hub 场景重构”，不是重写整个 Dungeon。
当前战斗地图、房间流程、Encounter、Boss 房等保持现状，不要大改。

一、总体设计目标

1. 保持游戏逻辑分辨率 320×240 不变。
2. 新增一个“大型可滚动地图”能力，优先用于 Hub / 基地。
3. Hub 不再是简单菜单，而是玩家可自由移动、可交互、可滚动浏览的大地图。
4. 采用“西幻风格主基地”设定，核心场景为：
   - 重生之泉（Spring of Rebirth）
   5. 玩家每次启动游戏、死亡返回、通关回归时，都会回到重生之泉附近。
   6. 角色选择也通过重生之泉完成。
   7. 其他现有系统以建筑 / 设施 / NPC 的方式嵌入基地地图。
   8. 当前 Dungeon 战斗地图仍保持固定屏幕、固定房间逻辑，不要强行改成滚动地图。
   9. 这次重构应建立通用“大地图基础设施”，未来可以扩展到其他非战斗地图或大型地图。

   二、产品与体验目标

   重构后的主流程应趋向于：

   Splash
   → Hub（主基地）
   → 在基地中移动和交互
   → 通过远征之门开始或继续 Run
   → Dungeon
   → Defeat / Victory
   → 回到 Hub 的重生之泉
   → 显示结算或继续准备下一次 Run

   不要再把“正式进入游戏后立刻看到传统首页菜单”作为主要体验。
   但为了兼容与调试，旧 TitleState 可以先保留作为备用入口，不要求本轮删除。

   三、主基地世界观与功能布局

   基地采用“西幻 / 古遗迹 / 安全圣所”风格。

   核心区域如下：

   1. 重生之泉（基地中央）
      功能：
         - 基地出生点
            - 死亡或通关后的回归点
               - 角色选择入口
                  - 可显示当前已选择角色
                     交互：
                        - 打开 Character Select / 角色选择 Overlay 或复用现有 CharacterSelectState
                           说明：
                              - 每次进入 Hub 时玩家默认出现在重生之泉附近
                                 - 每次失败、通关、返回 Hub，都回到这里
                                    - 这是基地最核心的视觉锚点

                                    2. 远征之门（基地南侧或中央偏下）
                                       功能：
                                          - Start New Run
                                             - Continue Run
                                                - Hard Mode
                                                   - Daily Challenge
                                                      - Expedition Confirm
                                                         交互：
                                                            - 打开 Expedition / 出击配置面板
                                                               - 允许开始或继续地牢冒险

                                                               3. 工匠庭院（基地西侧）
                                                                  功能：
                                                                     - Meta Upgrade
                                                                        - Refund / 重置成长
                                                                           - 局外成长相关内容
                                                                              交互：
                                                                                 - 可拆成 2~3 个交互点，如铁匠炉、附魔桌、重铸石
                                                                                    - 也可以先统一打开现有升级面板

                                                                                    4. 军械回廊（基地东侧）
                                                                                       功能：
                                                                                          - 武器展示
                                                                                             - 起始装备预览
                                                                                                - 未来初始武器配置扩展位
                                                                                                   当前最低要求：
                                                                                                      - 可作为简单交互区，先打开武器信息 / 图鉴页
                                                                                                         - 不要求本轮重做完整武器配置逻辑

                                                                                                         5. 试炼祭坛（基地东南）
                                                                                                            功能：
                                                                                                               - Hard Mode
                                                                                                                  - Daily Challenge
                                                                                                                     - 未来挑战模式
                                                                                                                        如果远征之门已经承载 Hard/Daily，则该区域可以先作为未来扩展区或挑战入口

                                                                                                                        6. 档案高塔（基地北侧）
                                                                                                                           功能：
                                                                                                                              - Records
                                                                                                                                 - Achievements
                                                                                                                                    - Codex / 图鉴
                                                                                                                                       - 历史 Run 记录
                                                                                                                                          交互：
                                                                                                                                             - 打开记录/图鉴/成就等现有界面

                                                                                                                                             7. 训练庭（基地东侧延伸）
                                                                                                                                                功能：
                                                                                                                                                   - 教程
                                                                                                                                                      - 武器试射
                                                                                                                                                         - 伤害测试
                                                                                                                                                            本轮可先做成保留区域或基础训练区，不要求完整实现复杂训练逻辑

                                                                                                                                                            8. 观星台 / 占星终端（基地东北）
                                                                                                                                                               功能：
                                                                                                                                                                  - Settings
                                                                                                                                                                     - 音量、显示、语言、控制设置
                                                                                                                                                                        交互：
                                                                                                                                                                           - 打开现有 Settings Overlay / SettingsState

                                                                                                                                                                           9. 其他区域
                                                                                                                                                                              - 静思花园 / 愿泉区：未来可承接 Wish Fountain
                                                                                                                                                                                 - 旅者营地：未来可承接商人 / 剧情 NPC
                                                                                                                                                                                    这些本轮可先占位，不要求完整功能

                                                                                                                                                                                    四、技术目标：建立大型地图基础设施

                                                                                                                                                                                    必须新增一套“可滚动世界地图”系统，但不要破坏现有战斗地图逻辑。

                                                                                                                                                                                    1. 保持当前战斗地图逻辑
                                                                                                                                                                                       - Dungeon 战斗房仍保持：
                                                                                                                                                                                            - 20 × 15
                                                                                                                                                                                                 - 16px tile
                                                                                                                                                                                                      - 固定视口
                                                                                                                                                                                                           - 固定房间切换
                                                                                                                                                                                                              - 不要把 Dungeon 全部改成滚动地图
                                                                                                                                                                                                                 - 不要破坏 DOOR_ZONES、Encounter、Room 流程、Minimap、房间状态持久化等

                                                                                                                                                                                                                 2. 新增通用世界地图能力（先用于 Hub）
                                                                                                                                                                                                                    应支持：
                                                                                                                                                                                                                       - 任意宽高 tile map
                                                                                                                                                                                                                          - 摄像机跟随
                                                                                                                                                                                                                             - 世界边界约束
                                                                                                                                                                                                                                - 多图层地图
                                                                                                                                                                                                                                   - 碰撞层
                                                                                                                                                                                                                                      - 交互对象层
                                                                                                                                                                                                                                         - 区域触发
                                                                                                                                                                                                                                            - 世界坐标与屏幕坐标转换
                                                                                                                                                                                                                                               - 可见区域裁剪绘制

                                                                                                                                                                                                                                               3. 摄像机系统
                                                                                                                                                                                                                                                  新增 Camera2D 或等价实现：
                                                                                                                                                                                                                                                     - viewport 固定为 320 × 240
                                                                                                                                                                                                                                                        - 摄像机跟随玩家
                                                                                                                                                                                                                                                           - 摄像机位置使用平滑跟随
                                                                                                                                                                                                                                                              - 摄像机最终渲染偏移必须 Math.round，保持像素稳定
                                                                                                                                                                                                                                                                 - 摄像机不能超出世界边界
                                                                                                                                                                                                                                                                    - 建议支持 dead zone（死区），减少画面抖动

                                                                                                                                                                                                                                                                    4. 渲染要求
                                                                                                                                                                                                                                                                       世界绘制和 UI 绘制必须分层：
                                                                                                                                                                                                                                                                          - camera.begin(ctx)
                                                                                                                                                                                                                                                                             - 绘制世界层（地图、建筑、NPC、玩家、特效、提示）
                                                                                                                                                                                                                                                                                - camera.end(ctx)
                                                                                                                                                                                                                                                                                   - 绘制 HUD / Overlay / 转场 / 调试信息

                                                                                                                                                                                                                                                                                   5. 世界地图裁剪绘制
                                                                                                                                                                                                                                                                                      不要每帧绘制整张大型地图。
                                                                                                                                                                                                                                                                                         必须根据摄像机可见区域，裁剪只绘制当前视口附近的 tile。

                                                                                                                                                                                                                                                                                         五、Hub 地图规格建议

                                                                                                                                                                                                                                                                                         本轮 Hub 地图建议：
                                                                                                                                                                                                                                                                                         - tileSize = 16
                                                                                                                                                                                                                                                                                         - 尺寸建议：80 × 60 tiles（约 1280 × 960）
                                                                                                                                                                                                                                                                                         - 可滚动，单张大地图
                                                                                                                                                                                                                                                                                         - 视口仍为 320 × 240

                                                                                                                                                                                                                                                                                         可以接受略有不同尺寸，但必须满足“大型可滚动地图”的目标，不要做成单屏。

                                                                                                                                                                                                                                                                                         六、地图结构与分层要求

                                                                                                                                                                                                                                                                                         建议至少支持这些图层：

                                                                                                                                                                                                                                                                                         1. ground
                                                                                                                                                                                                                                                                                            - 地砖、道路、草地、石板等

                                                                                                                                                                                                                                                                                            2. groundDetail
                                                                                                                                                                                                                                                                                               - 裂缝、草丛、符文、地面装饰

                                                                                                                                                                                                                                                                                               3. collision
                                                                                                                                                                                                                                                                                                  - 不可通行区域

                                                                                                                                                                                                                                                                                                  4. backObjects
                                                                                                                                                                                                                                                                                                     - 墙体后层、建筑主体后层

                                                                                                                                                                                                                                                                                                     5. ySortedObjects
                                                                                                                                                                                                                                                                                                        - 玩家、NPC、箱子、设施、小雕像等
                                                                                                                                                                                                                                                                                                           - 按 sortY 排序绘制

                                                                                                                                                                                                                                                                                                           6. upper / frontObjects
                                                                                                                                                                                                                                                                                                              - 屋檐、树冠、拱门前景、建筑前层

                                                                                                                                                                                                                                                                                                              如果当前美术资源有限，可以先用较简单的 tile / block 表现，但结构必须预留。

                                                                                                                                                                                                                                                                                                                       - 支持 walk / idle 动画

                                                                                                                                                                                                                                                                                                                                                                                          3. 交互提示
